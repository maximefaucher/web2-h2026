# Introduction à JavaScript

## 1. Exécuter JavaScript

JavaScript est un langage de programmation. À l'origine, c'était un langage **interprété**, mais aujourd'hui, on parle plutôt d'une **exécution via une compilation JIT (Just In Time)**. Donc il n'est ni 100% interprété, ni 100% compilé.

De nos jours, on utilise des **moteurs JavaScript** pour l'exécution du code JS. Les navigateurs modernes fournissent évidemment un moteur JavaScript :

|Navigateur|Moteur JS|
|---|---|
|Chrome|V8|
|FireFox|SpiderMonkey|
|Safari|JavaScriptCore|

Le navigateur utilise un moteur JS ainsi que des APIs Web telles que le `DOM` (Document Object Model), `fetch` (pour récupérer des données d'un serveur), `Web Storage`, etc.

Cependant, il existe une manière différente d'exécuter JavaScript sans nécessiter un navigateur : `Node.js`, qui est composé des éléments suivants :

- Le moteur **V8** de Chrome
- Une couche de C++
- Des APIs système

**Résultat** : JavaScript peut interagir avec la machine. À condition bien entendu que Node.js soit installé sur ladite machine!

### Différence navigateur vs Node.js (tableau comparatif)

|Critère|Navigateur|Node.js|
|---|---|---|
|Moteur JS|Oui (variable)|Oui (V8 de Chrome)|
|DOM|✅|❌|
|Accès aux fichiers|❌|✅|
|Réseau|Limité|✅|
|Objectif|Interface utilisateur|Serveur / scripts|
|Environnement|*Sandbox* (isolé et sécuritaire)|Système|

## 2. Déclaration de variables en JavaScript

Voici un résumé pratique sur la **déclaration de variables en JavaScript**, telle qu'on l'emploie aujourd'hui. On retrouve trois mots-clés pour la déclaration de variables en JS : `const`, `let` et `var`.

Comme JavaScript n'est pas un langage *typé*, **il n'est pas nécessaire de déclarer un type** lorsqu'on déclare une variable JavaScript.

|Mot-clé|Portée|Réaffectation|Usage recommandé|
|---|---|---|---|
|`const`|Bloc (dans les `{ }`)|Interdite\*|Par défaut|
|`let`|Bloc (dans les `{ }`)|Autorisée|Si la variable doit être réassignée|
|`var`|Fonction (globale sinon)|Autorisée|À ÉVITER|

\* La **mutation** est autorisée pour les objets/tableaux (les propriétés/éléments peuvent changer), même si l'objet/le tableau est déclaré `const`.

### Bonnes pratiques

- Prendre `const` de préférence, `let` si la réassignation est nécessaire, mais jamais `var` (sauf code *legacy*).
- Nommer clairement (camelCase pour variables/fonctions, SCREAMING_SNAKE_CASE pour constantes fixes comme `NB_QUESTIONS`).
- Limiter la portée (déclarez la variable au plus près de son usage).
- Préférer `const` pour déclarer les fonctions (évite la réassignation accidentelle).

## 3. Tableaux et objets en JavaScript

Les tableaux et les objets sont couramment utilisés en JavaScript.

Les **tableaux** sont des énumérations de valeurs entre `[ ]` et séparées par des `,`.

Les **objets** sont des énumérations de paires clé/valeur entre `{ }` et séparées par des `,`. Les clés sont généralement écrites en camelCase sans espaces / caractères spéciaux et elles sont suivies du caractère `:` qui précède la valeur. On appele la combinaison clé/valeur une **propriété** de l'objet.

### Exemples de tableaux et objets JavaScript

|Déclaration|Description|
|---|---|
|`const tab = [1, 3, 5, 2];`|Tableau de 4 entiers|
|`let valeurs = [false, "Bonjour", 2.5, [0, 0, 1]];`|Tableau de 4 valeurs diverses (dont un sous-tableau)|
|`const obj = {};`|Objet vide|
|`let monObjet = { nom: "Nick Suzuki", age: 25, tailleEnM: 1.80, estRetraite: false};`|Objet comprenant 4 propriétés|
